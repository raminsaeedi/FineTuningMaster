# Dashboard Design Fine-Tuning

Fine-tuning a small language model (`Qwen/Qwen2.5-0.5B-Instruct`) with **LoRA** so it can generate structured JSON dashboard design recommendations from a short text description.

**Master Thesis** | PEFT + RAG for Dashboard Design Quality

---

## What Does This Project Do?

You give the model a short description of a dashboard you need — called a **dashboard brief** — and the model returns a structured JSON recommendation that tells you:

- Which KPIs to show
- Which chart type fits each KPI (and why)
- How to arrange the layout
- Which colors and labels to use
- Which interactive features to add

**Example input (dashboard brief):**

```
Create a dashboard for a sales manager who wants to monitor monthly revenue,
sales by region, top products, conversion rate, and customer churn.
```

**Example output (structured JSON):**

```json
{
  "context_summary": "This Sales dashboard helps the manager track revenue trends and regional performance.",
  "kpi_task_chart_mapping": [
    {
      "kpi": "Monthly Revenue",
      "user_task": "trend over time",
      "recommended_chart": "line chart",
      "rationale": "Line charts communicate trends over time clearly."
    }
  ],
  "layout_hierarchy": "Top row: KPI cards. Middle: line chart. Bottom: bar chart by region.",
  "labels_scales_colors": "Blue-green palette for positive trends; red for negative deviations.",
  "interactions": [
    "Date range filter",
    "Region dropdown",
    "Drill-down on click"
  ],
  "design_rationales": [
    "KPI cards give immediate visibility.",
    "Consistent colors reduce cognitive load."
  ]
}
```

## Setup on Windows (Step by Step)

### 1. Open the project folder in VS Code

Open VS Code, then go to **File → Open Folder** and select the `dashboard-design-finetuning` folder.

Open the integrated terminal: **Terminal → New Terminal**

### 2. Create a virtual environment

```cmd
python -m venv .venv
```

This creates a folder called `.venv` that keeps all packages isolated from your system Python.

### 3. Activate the virtual environment

```cmd
.venv\Scripts\activate
```

You should see `(.venv)` at the start of your terminal prompt. You need to do this every time you open a new terminal.

### 4. Install required packages

```cmd
pip install -r requirements.txt
```

If you have an NVIDIA GPU and want to use it, also run:

```cmd
pip install torch --index-url https://download.pytorch.org/whl/cu121
```

## Running the Scripts (in order)

Run all scripts from inside the `dashboard-design-finetuning` folder.

### Script 01 — Check your environment

```cmd
python scripts/01_check_environment.py
```

**What it does:** Checks that Python, PyTorch, and all required libraries are installed correctly. Also tells you whether a GPU is available.

**Expected output:**

```
[OK] Python 3.11.x
[OK] PyTorch 2.x.x
[OK] CUDA available: NVIDIA GeForce RTX ...
[OK] transformers 4.x.x
[OK] peft 0.x.x
[OK] trl 0.x.x
```

---

### Script 02 — Generate synthetic training data

```cmd
python scripts/02_generate_synthetic_dataset.py
```

**What it does:** Creates 80 synthetic dashboard design examples across 10 domains (Sales, Finance, HR, Marketing, Logistics, Healthcare, Energy, Customer Support, E-Commerce, Project Management). Each example has a dashboard brief as input and a structured JSON recommendation as output.

**Output file:** `data/raw/synthetic_dashboard_recommendations.jsonl`

---

### Script 03 — Prepare dataset for training

```cmd
python scripts/03_prepare_dataset.py
```

**What it does:** Converts the raw JSONL into the instruction-tuning format that the model expects. Splits the data 90/10 into train and validation sets.

**Output files:**

- `data/processed/train.jsonl` — 72 examples
- `data/processed/validation.jsonl` — 8 examples

---

### Script 05 — Test the base model (before fine-tuning)

```cmd
python scripts/05_inference_base_model.py
```

**What it does:** Downloads `Qwen/Qwen2.5-0.5B-Instruct` and runs it on a fixed test brief **without any fine-tuning**. This is your baseline — you can compare it later against the fine-tuned version.

**Output file:** `outputs/predictions/base_model_prediction.json`

> **Note:** The model download is about 1 GB. It is cached locally after the first download.

---

### Script 04 — Fine-tune the model with LoRA

```cmd
python scripts/04_train_lora.py
```

**What it does:** Fine-tunes the base model using LoRA (a lightweight fine-tuning method that only trains a small number of extra parameters). Saves the trained adapter to `outputs/models/qwen-dashboard-lora`.

**LoRA settings used:**

- `r = 8` (rank — controls how many parameters are trained)
- `lora_alpha = 16`
- `target_modules = q_proj, v_proj`
- 1 epoch, learning rate 2e-4

**Expected duration:**

- GPU (e.g. RTX 3060): ~5–15 minutes
- CPU only: several hours — use Colab instead

**Output:** `outputs/models/qwen-dashboard-lora/`

---

### Script 06 — Test the fine-tuned model

```cmd
python scripts/06_inference_finetuned_model.py
```

**What it does:** Loads the base model + your saved LoRA adapter and runs inference on the same test brief as script 05. This lets you compare the fine-tuned model against the baseline.

**Output file:** `outputs/predictions/finetuned_model_prediction.json`

---

### Script 07 — Evaluate schema compliance

```cmd
python scripts/07_evaluate_schema_compliance.py
```

**What it does:** Checks whether the model outputs contain all required JSON fields with the correct types. Computes a **completeness score** (0.0 to 1.0) for each model.

**Terminal output example:**

```
Base model:
  valid_json        : True
  completeness_score: 0.5 (3/6 fields)
  missing_fields    : layout_hierarchy, interactions, design_rationales

Fine-tuned model:
  valid_json        : True
  completeness_score: 1.0 (6/6 fields)
  missing_fields    : none
```

**Output file:** `outputs/predictions/schema_compliance_report.json`

## Common Errors and Fixes

### CUDA not available

```
No GPU detected — running on CPU.
```

This is just a warning, not an error. The scripts will still run, but training (script 04) will be very slow. Use Google Colab for training if you don't have a GPU.

---

### Out of memory (CUDA out of memory)

```
RuntimeError: CUDA out of memory.
```

**Fix:** The batch size is already set to 1. If you still get this error, try closing other GPU applications, or use Google Colab which gives you a fresh GPU environment.

---

### bitsandbytes does not work on Windows

`bitsandbytes` is a library for 4-bit quantization (QLoRA). It has known issues on Windows. This project intentionally **does not use bitsandbytes** — it uses plain LoRA instead, which works fine on Windows without any extra setup.

---

### Hugging Face model does not download

```
OSError: We couldn't connect to 'https://huggingface.co'
```

**Fixes:**

- Check your internet connection.
- If you are behind a corporate firewall or VPN, try disabling it temporarily.
- You can also download the model manually from https://huggingface.co/Qwen/Qwen2.5-0.5B-Instruct and point the scripts to the local path.

---

### Script 06 fails because adapter is missing

```
FileNotFoundError: LoRA adapter not found at: outputs/models/qwen-dashboard-lora
```

**Fix:** Run script 04 first to train and save the adapter.

## Next Steps

Once you have a working fine-tuned model, here are ideas for improving it further:

1. **More synthetic data** — Increase from 80 to 500+ examples by adding more domain templates and KPI variations in `scripts/02_generate_synthetic_dataset.py`.

2. **Real datasets** — Explore publicly available chart and dashboard datasets:
   - [ChartQA](https://github.com/vis-nlp/ChartQA) — question answering over charts
   - [Chart-to-Text](https://github.com/vis-nlp/Chart-to-Text) — chart summarization
   - [NLV Corpus](https://github.com/nlvcorpus/nlvcorpus.github.io) — natural language to visualization

3. **RAG (Retrieval-Augmented Generation)** — Instead of relying only on what the model learned during training, add a retrieval step that fetches relevant design guidelines (e.g. from a PDF or knowledge base) and injects them into the prompt at inference time.

4. **Quantitative comparison** — Run both `05_inference_base_model.py` and `06_inference_finetuned_model.py` on a larger test set and compare completeness scores, JSON validity rates, and qualitative output quality.

5. **Longer training** — Try `num_train_epochs = 3` or `5` in `scripts/04_train_lora.py` to see if the model learns the schema more reliably.
